# 题目数据 Schema（标注产物的目标格式）

> 「打牌式」答题界面（`assets/mahjong-tiles/demo.html`）直接消费的题目格式。
> 建议**数据标注 agent 以此为产出目标**，两边对齐后即可无缝接入正式页。

## 牌编码
- `m` 万 · `p` 饼/筒 · `s` 索 · `z` 字
- `1-9m/p/s`；`1-7z` = 东 南 西 北 白 发 中
- `0m/0p/0s` = 赤五（红宝牌），与普通 `5x` **视为不同牌**
- `back` = 牌背

## 一道题（JSON）
```json
{
  "id": "Q01",
  "situation": {
    "round": "东",                 // 场风
    "kyoku": 1,                     // 第几局
    "honba": 0,                     // 本场数
    "seat":  "东",                  // 自风（座位风）
    "turn":  6,                     // 巡目
    "extra": "已碰「中」，有役"       // 可选：补充场况说明（自由文本）
  },
  "dora": ["5m"],                   // 宝牌指示牌（可多张；里宝不放这）
  "melds": [                        // 副露（无则空数组 []）
    { "type": "pon", "tiles": ["7z","7z","7z"], "from": "across" }
    // type: pon(碰) | chi(吃) | minkan(明杠) | ankan(暗杠)
    // from: left(上家) | across(对家) | right(下家)；ankan 可省略 from
  ],
  "concealed": ["3m","4m","0m","6m","7m","8m","2p","3p","9s","9s"],  // 门清手牌（不含自摸）
  "draw": "5s",                     // 自摸进来的那张（界面单独展示在最右）
  "canRiichi": false,               // 是否「门清听牌、可宣告立直」的局面
  "answers": [                      // 可接受答案；命中任一即判对
    { "action": "discard", "tile": "5s", "riichi": false }
  ],
  "why": "……解析，可含 <b> 标签"
}
```

## answer（答案）模型
每个答案是一个三元组：
- `action`：`discard`（切牌）| `ankan`（暗杠）
- `tile`：涉及的牌编码（discard = 被切的牌；ankan = 被杠的牌，用普通编码如 `5s`）
- `riichi`：该切牌是否**同时宣告立直**（仅 `canRiichi=true` 时有意义）

### 判定规则（界面已内置）
1. **同张等价**：手牌里多张相同编码的牌，点哪张都一样；**自摸牌与手牌同等**对待（按编码匹配，不看位置）。
2. **赤五区分**：`0x` ≠ `5x`，切赤五与切普通五是不同答案。
3. **暗杠 ≠ 普通切**：同一张牌，`{action:"ankan"}` 与 `{action:"discard"}` 是不同答案。
4. **立直 ≠ 默听**：门清听牌时，`riichi:true` 与 `riichi:false` 是不同答案。
5. 一题可有**多个**可接受答案（例：「切 9p 或 切 1z 都对」），都放进 `answers` 数组。
6. **立直合法性由前端自动判定**：界面内置听牌检测（标准型+七对+国士），宣告立直后只有「切掉后仍听牌」的牌可选、其余变暗——标注**无需**手动给立直白名单。
7. **务必收录所有牌效等价的最优解**：如某手切 1m/4m/7m 听同样的牌，三者都要进 `answers`，否则会误判。

## 每题需要标注的内容清单
1. **场况** situation（场风 / 局 / 本场 / 自风 / 巡目，+ 可选自由说明）
2. **宝牌指示牌** dora
3. **手牌拆三段**：副露 `melds` / 门清 `concealed` / 自摸 `draw`
4. **是否门清听牌** `canRiichi`
5. **正确答案集合** `answers`（注意特殊形：暗杠、立直/默听、赤五、多解）
6. **解析** `why`
